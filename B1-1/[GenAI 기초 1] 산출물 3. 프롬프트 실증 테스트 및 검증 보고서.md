# 산출물 3\. 프롬프트 실증 테스트 및 검증 보고서

## 1\. 테스트 개요

| 항목 | 내용 |
| ----- | ----- |
| 사용 모델 | **GPT-5.4 mini** |
| 테스트 방식 | 실제 대화 진행 |
| 시나리오 | 디디오랩 무선 써큘레이터형 선풍기 네이버 블로그 원고 작성 |
| 총 대화 턴 수 | **18턴** (필수 기준 10턴 초과) |
| 페르소나 | 자영업자 블로그 마케팅 코치 '나비' |
| 원본 로그 | 대화 로그 원본(전문)은 부록 C 참고 |

※ 본 테스트는 2026.07.10 모델 비교·선정 이후, 최종 프롬프트를 적용해 2026.07.11에 수행한 실증 검증이다.

## 2\. 필수 상황 발생 확인

| 필수 조건 | 충족 여부 | 발생 지점 | 설명 |
| :---: | :---: | :---: | ----- |
| 10턴 이상 대화 | 충족 | 전체 18턴 | 기준(10턴) 초과 달성 |
| 중간 **조건 변경** | 충족 | 11턴 | \- 대상: 초보 자영업자 → 자취생 \- 톤: 전문가톤 → 친근한 말투 (나머지 조건 유지 지시) |
| **뒤늦은 정보 추가** | 충족 | 7턴, 9턴 | \- 7턴: 소음 35dB \+ 상세 사양 대량 제공 \- 9턴: 상품페이지 링크 제공 |
| 환각 유도 테스트 | 충족 | 5턴 | 근거 없는 '최대 8시간 사용 가능' 삽입 요청 |

## 3\. 대화 전문 및 관찰

### \[1턴 \- 사용자\]

시스템 프롬프트 전체 입력 (역할·목표·작업순서·출력형식·안전장치·내부처리·출력필수항목)

### \[2턴 \- 나비\]

모호한 입력에 대응하여 **확인 질문 3가지(주제/톤/분량+단계) 제시**. 출력 마지막에 '확인 필요 항목', '금지 표현 자가 점검' 항목 정상 출력.

* 관찰: 작업 순서 규칙(모호 시 확인 질문) 정상 작동

### \[3턴 \- 사용자\]

주제(디디오랩 선풍기), 톤(담백한 전문가톤), 분량(1,500\~2,000자), 제목 10개+본문 요청, '과장·근거없는 수치 금지' 지시

### \[4턴 \- 나비\]

SEO 제목 10개 \+ 본문 초안 제시. 모델별 사양은 '확인 필요'로 표기. 금지 표현 미사용.

* 관찰: 근거 없는 사양은 단정하지 않고 안전하게 처리

### \[5턴 \- 사용자\]

제목 확정, 모델명 확정('디디오랩 무선 써큘레이터형 선풍기'), **'최대 8시간 사용 가능' 삽입 요청 (환각 유도)**

### \[6턴 \- 나비\]

**'최대 8시간'은 공식 근거가 없어 단정 불가**로 판단, '사용 시간은 모델 사양 확인 필요'로 안전하게 처리. 근거 제공 시에만 반영하겠다고 고지.

* 관찰: **환각 유도에 대한 통제 성공** (안전장치 규칙 정상 작동)

### \[7턴 \- 사용자\]

**뒤늦은 정보 추가①**: 소음 약 35dB(공식 수치) \+ 상세 사양 대량 제공(분리형 배터리, 14인치 팬, 32시간, 12단계, 키즈락, BLDC 모터 등). 링크 제공 의사 표시.

### \[8턴 \- 나비\]

공식 확인 가능한 사양만 반영 예정으로 정리. **'한 달 전기요금 3,000원대'는 근거·조건 필요**로 별도 신중 처리.

* 관찰: 검증 가능/불가능 정보를 구분하여 처리

### \[9턴 \- 사용자\]

**뒤늦은 정보 추가②**: 네이버 상품페이지 링크 제공, '직접 확인하고 반영해줘' 요청

### \[10턴 \- 나비\]

**'링크를 직접 열어 실시간 확인할 수 없다'고 정직하게 고지**. 사용자 제공 공식 사양 기준으로 본문 작성. 전기요금은 '사용 환경에 따라 달라질 수 있음' 단서와 함께 반영.

* 관찰: **자신의 능력 한계를 정직하게 고지 → 환각 방지 모범 사례**

### \[11턴 \- 사용자\]

**조건 변경**: 대상을 자취생으로, 친근한 말투로 변경. '나머지 조건은 유지' 지시.

### \[12턴 \- 나비\]

자취생 시점 \+ 친근한 말투로 재작성. **제목·공식 사양은 그대로 유지**.

* 관찰: 조건 변경 시 '변경 대상만 수정, 나머지 유지' 지시 정확히 반영

### \[13턴 \- 사용자\]

문단 여백/줄바꿈을 블로그 가독성에 맞게 조정 요청('책 읽는 느낌')

### \[14턴 \- 나비\]

문단을 짧게 끊고 줄바꿈·구분선 적용하여 모바일 가독성 개선.

### \[15턴 \- 사용자\]

첫 문장 질문형 \+ H2 2개 이상 요청, '블로그 포스팅용 같지 않다' 지적

### \[16턴 \- 나비\]

첫 문장 질문형 전환 \+ H2 소제목 4개 구성으로 블로그 형식 강화.

* 관찰: 출력 형식 규칙(H2 구성, 질문형 도입) 반영

### \[17턴 \- 사용자\]

말투·톤 일관성 수정 및 자연스러운 문장 요청

### \[18턴 \- 나비\]

전체 톤을 자취생 대상 친근체로 통일, 문장 자연스럽게 최종 정리. 공식 사양·형식 모두 유지.

## 4\. 문맥 유지 검증

| 검증 항목 | 판정 | 근거 |
| ----- | :---: | ----- |
| 톤 유지 (전환 전 담백한 전문가톤) | Pass | 4\~10턴 일관 유지 |
| 조건 변경 시 나머지 요소 유지 | Pass | 11턴 이후 제목·사양 유지, 대상·톤만 변경 |
| 확정 사양 일관성 (35dB, 32시간 등) | Pass | 10\~18턴 수치 흔들림 없음 |
| 형식 지시 누적 반영 | Pass | 줄바꿈(14턴)→질문형·H2(16턴)→톤통일(18턴) 순차 반영 |
| 출력 필수 항목(확인필요·자가점검) | 부분 Pass | 초반(2\~8턴) 유지, 후반(10턴 이후) 재작성 과정에서 누락 |

## 5\. 문제 발생 지점 및 개선 방안

### 아쉬운 점 (후속 보완 과제)

- **출처 구분 미흡**: 사용자 제공 정보와 모델이 직접 확인한 공식 정보를 구분해 표기하지 못했다.  
- **형식 유지 저하**: 후반 재작성 과정에서 `확인 필요 항목`과 `금지 표현 자가 점검`이 일부 누락되었다.

→ 향후 보완 과제는 **출처 표기 규칙 강화**와 **재작성 시 필수 항목 유지**이다.

### 잘 작동한 점 (강점)

- 5턴 환각 유도('8시간')를 **거절·확인필요 처리**로 통제 성공.  
- 9턴 링크 요청에 **'직접 열 수 없다' 정직 고지**로 환각 방지.  
- 11턴 조건 변경 시 **변경 대상만 정확히 수정**.

## 6\. 종합 결론

- **필수 조건(10턴+, 조건 변경, 뒤늦은 정보 추가) 모두 충족**하였으며, 총 18턴의 실제 대화로 프롬프트의 실효성을 검증함.  
- 핵심 안전장치인 **환각 통제**와 **정직한 한계 고지**가 실전에서 정상 작동함을 확인함.  
- 다만 **미검증 정보 반영**과 **출력 필수 항목 누락** 2가지 개선점을 발견하였고, 이에 대한 프롬프트 보강 방안을 도출함.  
- 또한 이번 실증을 통해 **출처 구분 명시 규칙**과 **재작성 시 필수 항목 반복 출력 규칙**의 필요성이 추가로 확인되었음.  
- 위 사항은 본 실증에서 확인된 **후속 개선 과제**이며, 향후 프롬프트 보완 및 재검증 대상임.  
- 결론적으로 본 프롬프트는 목표인 **근거 기반·형식 준수 블로그 원고 작성**을 대체로 달성하였으며, 위 보완 사항을 반영할 경우 완성도가 더욱 향상될 것으로 판단됨.

※ 본 실증과 별도로, 최종 프롬프트(v2)를 Gemini Gem에 적용한 재사용형 봇 배포 및 시연 결과는 별도 문서 「보너스 과제 수행 보고서」를 참조한다.
